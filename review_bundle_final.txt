## Key Architectural Decisions

### 1. Concurrency Model
- **Atoms + CAS:** Uses `compare-and-set!` with `loop/recur` for session management
- **No locks:** Removed blocking `locking` in favor of optimistic concurrency
- **In-flight gate:** `fetch-inflight?` atom prevents thundering herd on cache miss

### 2. Cache Strategy
- **TTL-based:** 60s throttle on `posts/update` check
- **Invalidate on write:** `add-bookmark!` and `delete-bookmark!` set `:checked-at 0`
- **Stale fallback:** Returns cached data on API errors

### 3. Rate Limiting
- **429 handling:** `pinboard-request` retries once after 5s sleep
- **Single retry:** Personal use bar (not full exponential backoff)

### 4. Session Safety
- **Pruning guarantee:** New session explicitly excluded from pruning pool
- **Dec max-sessions:** Keeps one slot guaranteed for new sid

### 5. Tag Filtering
- **Correct ordering:** Fetch all → filter by tag → apply limit (not the reverse)

### 6. Nix Module Structure
- **Top-level nixosModules:** Outside `eachDefaultSystem` for importability
- **Single-file src:** `src = ./pinboard_mcp.bb` with `dontUnpack = true`
- **DynamicUser:** Ephemeral system user for security
